The Domain Name System (DNS) is one of the most important services in computer networks. It is responsible for translating human-readable domain names such as google.com or youtube.com into numerical IP addresses that computers use for communication. Without DNS, users would need to remember complex IP addresses for every website they visit. DNS makes internet access faster, simpler, and more user-friendly.
As the internet has grown, DNS has become a critical part of almost every online activity, including web browsing, email communication, cloud services, and mobile applications. Since millions of DNS requests are processed every second, any attack on DNS infrastructure can affect performance, privacy, and security. Therefore, protecting DNS traffic has become an important topic in modern network security.
One common DNS-related cyberattack is DNS spoofing, also known as DNS cache poisoning. In this attack, a malicious user sends fake DNS responses so that a victim receives an incorrect IP address instead of the legitimate one. As a result, users may be redirected to phishing websites, fake login pages, or malicious servers without their knowledge. This can lead to credential theft, malware infection, privacy loss, and disruption of trusted communication.
To understand and address this problem, this project develops a DNS Spoofing Detection and Monitoring System using both simulation and practical implementation methods. The first part of the project creates a real-time dashboard that simulates spoofing behavior in a controlled environment. It monitors requested domains, tracks redirected requests, stores logs, and generates alerts whenever suspicious DNS traffic patterns are detected. This demonstrates how monitoring systems can help identify possible spoofing attacks.
The second part of the project provides a practical Linux-based DNS demonstration using Python, Scapy, and a custom local DNS server. Real DNS query packets are captured and analyzed, while selected domains are intentionally mapped to administrator-defined fake IP addresses in a controlled environment. This shows how DNS responses can be manipulated and how packet-level monitoring can detect such activities. The technologies used in this project include Python, Flask, Ubuntu (WSL), Scapy and Kali Linux, creating a hybrid model that combines software simulation, live packet analysis, and DNS security concepts.
